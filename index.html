Git and Git Hub: an Overview

What is Git?

Git is a distributed version control system used to track changes in files (usually source code). It helps developers save versions of their work, collaborate, and roll back to earlier states if needed.

Why Git is important

  • Tracks every change made to files
  • Allows multiple people to work on the same project
  • Helps recover previous versions of code

Git was created by Linus Torvalds in 2005.

What is GitHub?

GitHub is a cloud-based hosting platform for Git repositories. It allows you to store Git projects online, collaborate with others, review code, and manage projects. GitHub is a service provided by GitHub, owned by Microsoft.

Key Git Concepts

  1. Repository (Repo): A folder that Git tracks.
  2. Working Directory: The current project files you are editing.
  3. Staging Area: Where changes are prepared before committing.
  4. Commit: A snapshot of changes with a message.
  5. Branch: A parallel version of the project (e.g., main, feature1).

Git Workflow (Simple)

Edit → Stage → Commit → Push (to GitHub)

Commonly Used Git Commands

  1. git --version : Check Git Version
  2. git config --global user.name "Your Name" git config --global user.email "youremail@example.com" : Configure Git (First Time)
  3. git init : Initialize a Repository - Creates a new Git repository in the folder.
  4. git status : Check Status - Shows modified, staged, and untracked files.
  5. git add filename : Add Files to Staging Area
  6. git add . : . adds all files
  7. git commit -m "Initial commit" : Commit Changes - Saves a snapshot with a message.
  8. git log : View Commit History
  9. git branch branch_name : Create a Branch
  10. git checkout branch_name : Switch Branch
  11. git switch branch_name : Switch Branch
  12. git checkout -b branch_name : Create and Switch Branch
  13. git merge branch_name : Merge Branches
  14. git clone repository_url : Clone a Repository (from GitHub)
  15. git remote add origin repository_url : Connect Local Repo to GitHub
  16. git push origin main : Push Code to GitHub
  17. git pull origin main : Pull Latest Changes
  18. git fetch : Fetch Changes (without merging)
  19. git reset filename : Undo staged file
  20. git reset --soft HEAD~1 : Undo commit (soft)
  21. git reset --hard HEAD~1 : Undo commit (hard)
  22. git branch -d branch_name : Delete a Branch

GitHub Features

  • Repositories – Project storage
  • Pull Requests – Request to merge code
  • Issues – Bug tracking
  • Actions – CI/CD automation
  • Fork – Copy someone else’s repository
